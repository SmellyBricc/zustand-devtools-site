Get started

Install the extension. Chrome Web Store launch preparation is in progress; the extension has not been submitted yet. Until the listing lands, the source is on GitHub and loads as an unpacked extension. The beta pairs extension 1.1.0 with bridge 0.2.0, built from that same repository (the README's beta setup walks through it). npm still serves bridge 0.1.1, which speaks the old protocol and does not work with this extension, so don't install it for the beta; 0.2.0 will be published to npm before the public launch.

Open DevTools and click the Zustand tab. Registered stores and the Timeline appear live; the experimental Component Hooks view works with zero setup (not guaranteed to be Zustand state).

Want unlimited Trace Sessions? Every install includes three full preview sessions. Pro unlocks unlimited ones once checkout opens. During the private beta nothing is charged.

Questions people ask

Can I use it today?

The project is in private beta. Chrome Web Store launch preparation is in progress, but the extension has not been submitted yet, so it is not installable from the store. Checkout is closed and no payments are accepted. The source is on GitHub and loads as an unpacked extension, and you can join the beta.

What do I need for it to work?

Chrome (or a Chromium browser that supports DevTools extensions) and an app using React 16.8+ with Zustand 4 or 5. Accurate store inspection uses the one-line zustand-devtools-bridge wrapper per store; the experimental Component Hooks view is zero-setup but not guaranteed to show Zustand state. During the beta, build the bridge from the repository: the npm release is still 0.1.1, which does not work with extension 1.1.0.

Does it conflict with the React DevTools extension?

No. It chains onto the same renderer hook React DevTools uses instead of replacing it, so both extensions work side by side regardless of which loads first.

What data does it collect?

No analytics and no telemetry. Application state stays on your machine: it flows from the inspected page to your own DevTools panel, and what the extension keeps (recent Trace Sessions, the free-preview counter, a license record) lives in your browser's local extension storage. The bridge keeps a bounded per-tab history in the inspected page's own sessionStorage. The only network requests are ones you trigger yourself: activating a license sends the key and a generic instance name to Lemon Squeezy. Full details in the privacy policy.

How does the license work?

Once checkout opens: one €9.99 purchase, Lemon Squeezy generates and delivers a license key, you paste it into the extension, and it activates through their License API. Planned allowance of up to 5 device activations, no subscription, no expiry. Live purchasing is not enabled during the private beta.

What if it doesn't work for me?

Email within 14 days of purchase for a full refund, no questions asked. Bug reports are welcome on GitHub issues either way. See terms and refunds.
