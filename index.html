The free tier reads your app the way React DevTools does.

No wrapper, no middleware, no import. The extension installs a renderer hook before React loads and walks the Fiber tree directly — so it works on any Zustand app you open, including ones you don't own.

State never leaves your machine. Data flows from the inspected page to your own panel and nowhere else — no server, no analytics, no telemetry.

Rewind the bug.

The action log is the paid tier: a named history of every state change, the diff between each step, and one-click time-travel — across all your stores in a single timeline. One line to opt a store in, mirroring Zustand's own middleware shape: 

import { create } from 'zustand';
import { withDevtoolsBridge } from 'zustand-devtools-bridge';

const useCartStore = create(
  withDevtoolsBridge(
    (set) => ({
      items: [],
      addItem: (item) =>
        set((s) => ({ items: [...s.items, item] }),
            false, 'addItem'),  // ← named in the log
    }),
    { name: 'cart' }
  )
);

Questions people ask

What do I need for it to work?

Chrome (or a Chromium browser that supports DevTools extensions), an app using React 16.8+ and Zustand 4+. The free inspector needs zero code changes; the action log needs the one-line zustand-devtools-bridge wrapper on each store you want logged.

Does it conflict with the React DevTools extension?

No — it chains onto the same renderer hook React DevTools uses instead of replacing it, so both extensions work side by side regardless of which loads first.

What data does it collect?

None. Component state never leaves your machine — it flows only from the inspected page to your own open DevTools panel. The single network request the extension can make is validating a license key you paste in. Full details in the privacy policy.
